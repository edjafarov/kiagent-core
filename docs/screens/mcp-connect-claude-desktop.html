Connect Claude Desktop
Remote · active

Claude Desktop can talk to KIAgent two ways. The custom-connector UI uses your remote HTTPS URL and works from any Claude Desktop session — even on another machine. The JSON config is the older path; it points at the local URL and only works on this Mac.

Option 1 — Add a custom connector (remote)
Recommended
  1. 1 In Claude Desktop, open Settings → Customize and Connectors.
  2. 2 Click Add custom connector.
  3. 3 Paste the URL below into the form. Claude Desktop will run the OAuth flow when you first call a tool.
  4. 4 KIAgent's tools appear in the tools menu. Verify by asking: "list my MCP tools".
https://alice-7k2n4q9p.lokalkiagent.de/mcp
Option 2 — Edit claude_desktop_config.json (local)
  1. 1 Open ~/Library/Application Support/Claude/claude_desktop_config.json in your editor. Create the file if it doesn't exist.
  2. 2 Merge this entry into the mcpServers object (or paste the whole snippet if the file is empty).
  3. 3 Quit and reopen Claude Desktop.
claude_desktop_config.json
{
  "mcpServers": {
    "kiagent": {
      "url": "http://127.0.0.1:7421/mcp",
      "headers": {
        "Authorization": "Bearer ••••••••"
      }
    }
  }
}
The bearer token in the JSON snippet is masked above — Copy snippet copies the real value to your clipboard. Treat it like a password. The remote URL has no token; Claude Desktop authenticates via OAuth on first call.
