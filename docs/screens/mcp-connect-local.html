Use Local URL directly
Using: ● Local

For MCP clients that just want a URL plus an optional auth header — pipe it through your own config file or paste it into the client's MCP settings page. The local endpoint is reachable from this machine only; for hosted clients (Claude.ai, mobile) set up Remote MCP from the hub.

  1. 1 Pick where your client reads MCP server config from — typically a JSON file like ~/.kiagent/mcp.json or the client's own settings panel.
  2. 2 Paste the snippet below as a server entry. The bearer header is required — the local server rejects unauthenticated requests.
  3. 3 Restart the client process so it picks up the new entry.
Suggested path ~/.kiagent/mcp.json
Server entry
{
  "url": "http://127.0.0.1:7421/mcp",
  "headers": {
    "Authorization": "Bearer ••••••••"
  }
}
Loopback only: 127.0.0.1:7421 is not exposed on your LAN. KIAgent must be running for the URL to respond.
